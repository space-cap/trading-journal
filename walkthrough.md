# 트레이딩 저널 개발 완료 보고서 (Walkthrough)

트레이딩 저널 애플리케이션의 MVP(최소 기능 제품) 개발을 성공적으로 완료했습니다.

## 1. 구현된 기능
### 백엔드 (Spring Boot + Kotlin)
- **Trade 엔티티**: 매매 상세 정보(종목명, 가격, 수량, 수수료, 진입 이유, 날짜 등) 저장 설계.
- **Trade 리포지토리**: 데이터베이스 작업을 위한 JPA 리포지토리.
- **Trade 서비스**: 매매 생성, 수정, 조회를 위한 비즈니스 로직.
- **Trade 컨트롤러**: REST API 엔드포인트 (`GET /api/trades`, `POST /api/trades` 등).
- **손익(P&L) 계산**: 엔티티 내에서 실현 손익 자동 계산 로직 구현.

### 프론트엔드 (React + Vite + TypeScript)
- **Tailwind CSS**: 스타일링 환경 구성.
- **매매 입력 폼**: 신규 매매 기록을 추가하기 위한 입력 폼.
- **매매 리스트**: 매매 내역을 표 형태로 조회 (손익에 따른 색상 강조).
- **대시보드 차트**: `recharts`를 사용하여 누적 수익 곡선 시각화.
- **프록시**: `/api` 요청을 백엔드로 전달하도록 설정.

## 2. 실행 방법
### 백엔드
1. IntelliJ IDEA 또는 터미널에서 프로젝트 열기.
2. `backend` 디렉토리로 이동.
3. 애플리케이션 실행:
   ```bash
   ./gradlew bootRun
   ```
   (참고: `gradlew`가 없다면 `gradle bootRun`을 사용하거나 IDE에서 `TradingJournalApplication.kt`를 실행하세요).

### 프론트엔드
1. 새 터미널 열기.
2. `frontend` 디렉토리로 이동.
3. 의존성 설치 (아직 안 했다면):
   ```bash
   npm install
   ```
4. 개발 서버 시작:
   ```bash
   npm run dev
   ```
5. 브라우저에서 `http://localhost:5173` 접속.

## 3. 검증
- **빌드**: 프론트엔드 빌드 성공 (`npm run build`).
- **린트**: 코드 린트 검사 통과.
- **기능 동작**:
    - 폼을 통해 매매 기록 추가.
    - 리스트에서 추가된 매매 내역 확인.
    - 차트 업데이트 확인.
