# 트레이딩 저널 구현 리포트

## 1. 프로젝트 개요
사용자의 매매 기록을 체계적으로 관리하고, 실현 손익(P&L)을 자동으로 계산하며, 시각화된 차트를 통해 성과를 분석할 수 있는 **개인 맞춤형 주식 매매 일지(Trading Journal)** 웹 애플리케이션의 MVP(Minimum Viable Product)를 구현했습니다.

## 2. 기술 스택 (Tech Stack)
### Backend
- **Language**: Kotlin (JDK 21)
- **Framework**: Spring Boot 3.2.0
- **Database**: H2 (In-memory, 개발용)
- **ORM**: Spring Data JPA
- **Build Tool**: Gradle (Kotlin DSL)

### Frontend
- **Language**: TypeScript
- **Framework**: React 19
- **Build Tool**: Vite
- **Styling**: Tailwind CSS 3.x
- **UI Library**: Recharts (차트 시각화)

## 3. 구현 기능 상세
### 3.1 백엔드 (API Server)
*   **Trade 엔티티 설계**:
    *   `symbol` (종목명), `entryPrice` (진입가), `quantity` (수량), `fee` (수수료), `reason` (진입 이유) 등의 필수 데이터 저장.
    *   `realizedPnl` (실현 손익): 매도(`exitPrice`) 발생 시 자동으로 손익을 계산하는 로직 포함.
*   **REST API 구현**:
    *   `POST /api/trades`: 신규 매매 기록 생성.
    *   `GET /api/trades`: 전체 매매 기록 조회.
    *   `PUT /api/trades/{id}`: 매매 기록 수정 (매도 처리 등).
    *   `DELETE /api/trades/{id}`: 매매 기록 삭제.
*   **서비스 계층**: 트랜잭션 관리 및 비즈니스 로직 처리.

### 3.2 프론트엔드 (User Interface)
*   **매매 입력 폼 (TradeForm)**:
    *   사용자가 직관적으로 매매 정보를 입력할 수 있는 UI 제공.
    *   API와 연동하여 데이터 저장.
*   **매매 리스트 (TradeList)**:
    *   저장된 매매 내역을 테이블 형태로 조회.
    *   손익(P&L)이 양수일 경우 초록색, 음수일 경우 빨간색으로 표시하여 시인성 확보.
*   **대시보드 차트 (DashboardChart)**:
    *   `Recharts` 라이브러리를 활용하여 누적 수익금 추이를 라인 차트로 시각화.
*   **개발 환경 설정**:
    *   Vite Proxy 설정을 통해 CORS 문제없이 백엔드 API와 통신 가능 (`/api` -> `localhost:8080`).

## 4. 프로젝트 구조
```
trading-journal/
├── backend/                # Spring Boot 프로젝트
│   ├── src/main/kotlin/    # Kotlin 소스 코드 (Controller, Service, Repository, Entity)
│   └── build.gradle.kts    # Gradle 빌드 설정
├── frontend/               # React 프로젝트
│   ├── src/
│   │   ├── components/     # UI 컴포넌트 (TradeForm, TradeList, DashboardChart)
│   │   ├── types/          # TypeScript 타입 정의
│   │   └── App.tsx         # 메인 페이지
│   ├── tailwind.config.js  # Tailwind CSS 설정
│   └── vite.config.ts      # Vite 및 Proxy 설정
└── docs/                   # 프로젝트 문서 (개발계획서, 구현리포트 등)
```

## 5. 실행 방법
1.  **백엔드**: `backend` 폴더에서 `./gradlew bootRun` 실행.
2.  **프론트엔드**: `frontend` 폴더에서 `npm install` 후 `npm run dev` 실행.
3.  **접속**: 브라우저에서 `http://localhost:5173` 접속.
